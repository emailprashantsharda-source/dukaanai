Common questions about Restaurants POS

Real questions from restaurants owners using HisaabNow.

How does table-wise billing work?
Set up your floor plan with table numbers. Take orders by table — each table holds an open bill that grows as courses arrive. Multiple waiters can add to the same table. When the customer asks for the bill, hit "close" and it prints. Critical feature for any serious restaurant POS app.
Can I split bills between customers at the same table?
Yes. Two ways: even split (4 friends, ₹2,000 bill, ₹500 each automatic) or item-by-item split (Rohit ordered the biryani, Priya the dal). The app prints separate receipts for each portion. Common for groups paying separately on UPI.
How does kitchen ticket printing work?
Connect a Bluetooth thermal printer in the kitchen. As soon as a waiter taps an order, the ticket prints in the kitchen with table number, items, and any modifiers ("no chili," "extra spicy"). Counter receipt prints separately when the customer pays. No more shouting orders, no lost slips.
Does Munim flag dishes that are losing money?
Yes — this is one of HisaabNow's strongest features. Set ingredient costs once, define each dish's recipe. If paneer prices go up and your paneer tikka recipe now costs ₹185 to make but you sell at ₹180, Munim flags it: "Paneer tikka below cost — review price." Catches problems before they kill your month.
Can I handle takeaway and dine-in together?
Yes. Switch any order to takeaway with one tap — no table assignment, just a phone number for the customer. The bill, kitchen ticket, and reports separate dine-in vs takeaway revenue. Useful for understanding which channel is actually earning you money.
What about Swiggy and Zomato order integration?
For now, log Swiggy/Zomato orders manually as a separate revenue channel — many restaurants do this and it works. Direct API integration with Swiggy/Zomato is on our roadmap. Reports separate dine-in, takeaway, Swiggy, and Zomato revenue so you see which channel earns more after their commission.
